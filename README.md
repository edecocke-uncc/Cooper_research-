# Sorghum Pan-Transcriptome Pipeline

Builds a 13-accession pan-transcriptome for bioenergy sorghum from per-accession genome assemblies and pan-gene annotations. Outputs a non-redundant CDS FASTA (one sequence per gene locus) and a presence/absence variation (PAV) matrix.

Based on the CP-NAM population described in [Voelker et al. 2023](https://doi.org/10.3389/fpls.2022.1040909).

---

Assembly Provenance
Three different assembly projects were used. Assemblies were generated by different methods and are documented separately below.
CP-NAM Parents (10 accessions)
Grassl, ChineseAmber, Leoti, Rio, PI229841, PI297155, PI329311, PI506069, PI510757, PI655972
Assemblies generated by Voelker et al. 2023 (PRJEB55613):
Sequencing: PacBio Sequel I long reads
Assembly: Canu v2.1.1 (Flye v2.9 for Grassl due to reduced read coverage)
Scaffolding: RagTag v2.1.0 against BTx623 v3.1 with minimap2; parameters -r -g 1 -m 10000000
Result: Chr01_RagTag chromosome naming convention
BTx623
Assembly: JGI v5.0.0 / SorghumBase v5.1 release (June 2023)
Source: Phytozome via SorghumBase release 6
Chromosome naming: Plain numeric (1, 2, 3...)
Note: BTx623 v5.1 is a higher quality assembly than v3.1 which was used as the RagTag scaffold reference for the CP-NAM parents. These are the same organism but different assembly versions. For pan-transcriptome CDS extraction this is not an issue as each accession uses its own matched GFF.
TX430
Sequencing: Oxford Nanopore MinION + BioNano Genomics DLS optical maps
Assembly: SMARTdenovo + Canu v1.6; chromosome-scale scaffolds
Source: Corteva Agriscience; Deschamps et al. 2018 (GCA_003482435.1)
Chromosome naming: ENA contig-style (QWKM01...); renamed for pipeline use
PI300119
Assembly source TBD — uses Chr01_RagTag naming suggesting RagTag scaffolding but not found in PRJEB55613

## Pipeline Steps

### Step 1 — Rename Chromosomes (Leoti and Rio)
Ware Lab GFFs use `Chr01_RagTag` chromosome naming. Leoti and Rio assemblies use plain numeric names (`1, 2, 3...`) and must be renamed before CDS extraction.

```bash
zcat input.fasta.gz | awk '
  /^>/ {
    name = substr($1, 2)
    if (name ~ /^[0-9]+$/) {
      printf ">Chr%02d_RagTag\n", name+0
    } else {
      print $0
    }
    next
  }
  { print }
' > output_renamed.fasta
```

### Step 2 — Filter BTx623 GFF
The full BTx623 GFF contains scaffold features that cause a Liftoff crash. Remove them:

```bash
grep -v "^#" btx623.pan-gene.aed_lt1.gff | \
  awk '$1 !~ /scaffold/' | \
  cat <(grep "^#" btx623.pan-gene.aed_lt1.gff) - \
  > btx623_chr_only.gff
```

### Step 3 — Liftoff Annotation for Grassl
Grassl has no Ware Lab annotation. Gene models are lifted from BTx623 v5.0.0:

```bash
liftoff \
    -g btx623_chr_only.gff \
    -o Grassl.pan-gene.liftoff.gff \
    -u Grassl.unmapped.txt \
    -a 0.95 -s 0.95 -copies -cds -polish -p 20 \
    Grassl.fasta \
    Sorghum_BTX623_5.1.dna.toplevel.fa
```
574 genes were unmapped and excluded.

### Step 4 — CDS Extraction (gffread)
Run as a SLURM array across all 13 accessions. See `gffread_array.sh`.

```bash
gffread <GFF> -g <FASTA> -x <OUTPUT>.cds.fasta
```

### Step 5 — Build PAV Matrix
Extracts gene IDs from `gene` features in each GFF and builds a binary presence/absence matrix.

```bash
python build_pav_from_gffs.py
# Output: PAV_13accessions.csv (61,583 loci × 13 accessions)
```

One Liftoff split-gene artifact was removed:
```bash
grep -v "split_gene_SORBI_3010G233900-g364" \
    PAV_13accessions.csv > PAV_13accessions_final.csv
# Final: 61,582 loci
```

### Step 6 — Aggregate Pan-Transcriptome FASTA
For each pan-gene locus, selects the longest CDS sequence across all 13 accessions. Uses GFF `Parent=` field to map transcript IDs to gene IDs.

```bash
python aggregate_pan_transcriptome.py
# Output: pan_transcriptome.fasta (61,582 sequences)
```

### Step 7 — Generate Pseudo-Annotation GTF
Creates a GTF for STAR with one `gene` and one `exon` feature per CDS sequence, spanning the full sequence length.

```bash
python make_pseudo_gtf.py
# Output: pseudo-annotation.gtf (61,582 gene + exon pairs)
```

### Step 8 — Build STAR Index

```bash
STAR --runMode genomeGenerate \
     --genomeDir star_index/ \
     --genomeFastaFiles pan_transcriptome.fasta \
     --genomeSAindexNbases 11 \
     --limitGenomeGenerateRAM 50000000000 \
     --runThreadN 16
```
